n and m are the actual sizes of the moves
a,b are the counts (may be larger)

1. n, or m means not zero. 
2. Its always true that n>=m.  
3. Always true that a>=n, b>=m.
4. Also, a>=b.

If not in quissence, then a=n, b=m. SO lets make the table assumming quissence.

   n     a     m     b                  meaning
--------------------------------------------------
   0     0     0     0               "end of game"
   0     0     0     b               IMPOSSIBLE   breaks rule 4: a >= b
   0     0     m     0               IMPOSSIBLE
   0     0     m     b               IMPOSSIBLE

   0     a     0     0               NOT a end of game
   0     a     0     b               NOT a end of game
   0     a     m     0               IMPOSSIBLE
   0     a     m     b               IMPOSSIBLE  breaks rule 2: n >= m 

   n     0     0     0               IMPOSSIBLE       
   n     0     0     b               IMPOSSIBLE
   n     0     m     0               IMPOSSIBLE         
   n     0     m     b               IMPOSSIBLE

   n     a     0     0               "end of game"    like a pinned pawn trying to capture.             
   n     a     0     b               not an end of game
   n     a     m     0               IMPOSSIBLE
   n     a     m     b               not an end of game
